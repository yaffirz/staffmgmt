## What changed
- **Models** (`models/models.py`): added `"TRANSFER"` to `ALLOWED_STATUS_ACTIONS`;
  added `AppSettings` table (`setting_id, tenant_id, key, value`, unique
  `(tenant_id, key)`) for feature toggles. Created by `create_all` on startup
  (non-destructive).
- **App settings** (`core/app_settings.py`, new): `get_setting` / `get_bool` /
  `set_setting` with a `DEFAULTS` map (`area_managers_can_move = "true"`).
- **Settings API** (`routes/settings.py` + `schemas/settings.py`, new):
  `GET/PATCH /api/v1/settings/{key}`, Admin/Super Admin only, audited on change.
- **Cluster API** (`routes/cluster.py` + `schemas/cluster.py`, new):
  - `GET /api/v1/cluster` — AM-only read view; stores in the AM's brands, staff
    per store, `also_covers` flag for additional-store staff; managers/supervisors
    sorted first.
  - `POST /api/v1/cluster/employees/{id}/move` — AM-only; gated by the toggle;
    validates primary-store-in-cluster + destination-in-cluster; sets
    `primary_store_id`; writes `staff_status_log` (TRANSFER), `notifications`
    (STAFF_MOVED → Admin), `audit_logs`.
  - `GET /api/v1/cluster/employees/search?name=` — AM/Admin/HR/Super Admin;
    case-insensitive contains across ALL tenant staff; capped 25; returns brands
    + stores per match.
  - `POST /api/v1/cluster/employees/{id}/request-assignment` — AM-only; no
    employee change; writes `notifications` (STAFF_REQUESTED → Admin) + `audit_logs`.
- **Seed** (`seed.py`): idempotently seeds `app_settings` defaults.
- **Wiring** (`main.py`): registered `cluster` and `settings` routers.
