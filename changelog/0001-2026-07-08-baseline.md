## Changelog conventions (read this first)
- One new `.md` file per committed change, saved in `/changelog`.
- Filename: `NNNN-YYYY-MM-DD-short-title.md` — `NNNN` is a zero-padded sequence.
  The highest number is the most recent entry.
- Each entry states: date, type, what changed (backend/frontend/db), files touched,
  any DB migration commands, and how to test.
- Never edit or delete prior entries.

## Project state as of this baseline

### Stack & environment
- Backend: FastAPI + SQLModel + PostgreSQL, in Docker Compose.
  - Start: `docker compose up -d` (add `--build` only after backend code/dep changes).
  - API http://localhost:8000, health `/health`, docs `/docs`.
  - DB inspect: `docker compose exec db psql -U staffadmin -d staffmgmt -c "..."`.
- Frontend: Flutter web, runs from `C:\Projects\staffmgmt\staff_frontend`.
  - Start (plain cmd window):
    `flutter run -d web-server --web-port 5000 --dart-define=API_BASE_URL=http://localhost:8000`
  - Canonical source folder is `staff_frontend\lib` (the ONLY frontend lib).

### Features complete
- Auth + role-based access (Super Admin, Admin, HR, Area Manager).
- Employees: create wizard, list (search, pagination, copyable cells, reviewed
  toggle, inline MAG edit), edit-via-wizard, delete, additional stores.
- Configurable new-hire form (Form Settings) + backend required-field enforcement.
- Brands & Stores hub: brands/stores/positions list, add, edit, bulk CSV import,
  multi-select delete with referential-integrity blocking.
- Bulk import for brands, stores, positions, AND employees — with downloadable CSV
  template, browser file upload, paste option, and bulleted instructions.
- Users & Roles (Admin/Super Admin): create/list/edit/delete user accounts.
  - Fields: username, email (required + unique per tenant), password + confirm,
    role. Area Manager role reveals a multi-brand picker.
  - Full edit (username/email/role/password/brands). List grouped by role.
- Area Manager "My Cluster" READ view (Phase 2a): store cards scoped to the AM's
  brands; staff grouped by role (managers/supervisors first), names indented;
  "also covers" tag for additional-store staff. Move/Request buttons are present
  but stubbed ("coming next").

### Database notes
- Tables incl.: users (now with `email` + unique index `uq_users_tenant_email`),
  area_managers, area_manager_brands (AM↔brand scoping), area_manager_stores
  (present, currently unused), employees, brands/stores/positions, countries,
  employee_additional_stores, staff_notes, staff_status_log, notifications,
  form_field_config, audit_logs.
- `employees` has NO employment_status column yet (needed for Terminate — planned).
- Migrations applied non-destructively via `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`
  and `CREATE UNIQUE INDEX IF NOT EXISTS` (no data wipes).
